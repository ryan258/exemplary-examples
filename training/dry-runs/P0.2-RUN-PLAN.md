## Independence protocol

Use separate, fresh agent sessions for Keystone, Verity, and Counterpoint.

1. Northstar creates a run ID and copies the candidate-only materials into the run record.
2. Forge gives Keystone only the published candidate brief, permitted tools, work envelope, and requested artifacts.
3. Keystone's complete output is frozen before assessment.
4. Verity and Counterpoint receive the same frozen evidence bundle and rubric in separate sessions. Neither receives the other assessor's score, rationale, or conversation history.
5. Northstar records both initial decisions before calibration.
6. The assessors then receive both evidence-cited score sheets and reconcile any rating difference greater than one point or any disagreement about criticality.
7. The Accountable Principal reviews only after the independent record is complete.

The same model family may fill different roles during the bootstrap, but shared context, copied reasoning, or one session pretending to be independent does not satisfy the two-assessor rule.

## Work envelope

### Allowed

- Read the curriculum version and candidate materials named below.
- Create synthetic briefs, registers, exhibits in Markdown, quality reviews, and work logs.
- Use public facts only if the exact source is retrieved and recorded; synthetic facts are preferred.
- Stop and ask Forge for a missing instruction or ambiguity.

### Prohibited

- External outreach, publishing, deployment, or client communication.
- Real personal, applicant, prospect, or client information.
- Inventing a source, citation, customer, credential, or observed result.
- Reading facilitator hidden facts before the relevant simulation event.
- Changing the rubric or self-certifying.

### Stop conditions

- A critical-failure trigger occurs.
- Required input or authority is missing.
- The candidate encounters data beyond D1.
- Candidate and facilitator materials cannot be separated reliably.

## Accessibility accommodation under test

All event prompts are delivered as written text with explicit timestamps rather than relying on audio, rapid speech, or a live performance cadence. Timing measures workflow decisions, not typing speed. The candidate may request a structured table in place of prose without changing the competency standard.

## Completion decision

The run is **complete** when every file exists and both assessors recorded initial judgments independently. It is **successful** only when:

- the minimum viable route provides evidence for every competency it claims to measure;
- no unresolved critical truth, confidentiality, authorization, or accessibility defect remains;
- assessor disagreement is within the published tolerance after calibration;
- the dry-run report identifies timing, confusing instructions, hidden context, missing artifacts, and curriculum changes;
- any authorization decision is configuration-specific and principal-ratified where external authority is proposed.

If the run is incomplete, repeat only the invalid stage. If it is complete but unsuccessful, revise the curriculum version and rerun the affected evidence route before P0.3.
