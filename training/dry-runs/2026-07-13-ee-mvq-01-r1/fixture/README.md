# EE-MVQ-01 Fixture Control

**Fixture version:** EE-MVQ-01

**Curriculum target:** EE-TRAINING-2026.07-draft

**Use:** P0.2 minimum viable qualification dry run

**Data:** Purpose-built synthetic D0/D1 only

This fixture makes the first qualification route executable without real client data or undocumented founder context. It is evaluation material, not evidence that a run occurred.

## Release boundaries

| File | Release to | Release point |
| --- | --- | --- |
| `01-CANDIDATE-PACKET.md` | Candidate and facilitator | Start of run |
| `02-FACILITATOR-PACKET.md` | Facilitator and program owner only | Before run; staged events released as directed |
| `03-ASSESSOR-KEY.md` | Assessors and program owner only | After candidate output is frozen |

The candidate must not receive facilitator or assessor material. Assessors must not receive one another's initial scores. A run is invalid if these boundaries cannot be demonstrated.

## Integrity

Copy the files into a run-specific directory and record their hashes or immutable versions in the run manifest. Do not edit the fixture during a run. Record discovered ambiguity in the dry-run report and revise the fixture only under a new version.
